AI Agent 工程结构

理解 AI Agent 的工程架构，核心在于理清提示词在系统中的流转生命周期、模型运行时的边界控制，以及内部循环与外部系统交互的确定性机制。

memory

Agent 是运行时

Agent 并非单纯的模型调用，而是一个包含状态管理、执行循环和边界校验的完整软件运行时环境。

psychology

模型只做决策

LLM 作为推理引擎，仅负责意图识别和动作规划，不直接执行任何可能改变系统状态的外部操作。

sync_problem

循环必须有退出条件

为了防止无限死循环，Agent loop 必须在设计阶段硬编码超时、最大步数或明确的终止指令。

hub

Skill 和 MCP 不是一回事

Skill 是 Agent 拥有的具体能力实现，而 MCP (Model Context Protocol) 是标准化外部系统接入的协议标准。

2. Agent loop 的内部结构

退出条件 触发场景 工程处理
Final Answer 模型认为任务已完成，输出最终结果 解析输出，结束循环，返回给用户
Max Steps 循环次数达到预设硬上限 强制熔断，返回部分结果或超时警告
Timeout/Budget 执行时间或 Token 消耗超出预算 阻断网络请求，抛出 ResourceExhausted 异常
Approval Denied 高风险操作（如删库）被人工拒绝 将拒绝信息注入上下文，交还给模型重新决策
Fatal Error 工具执行崩溃或外部 API 严重异常 捕获异常，记录错误日志，进入降级策略
